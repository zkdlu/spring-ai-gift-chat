# spring-ai-gift-chat

# 기능 요구 사항
사용자가 자연어로 선물 고민을 이야기하면 대화를 통해 적절한 선물을 추천해주는 AI 기반 선물 추천 서비스를 구현한다.

사용자는 자연어로 선물 추천을 요청할 수 있다.
예: "친구 생일 선물 추천해 줘"
서비스는 선물 추천 도우미 역할로 사용자의 메시지에 응답한다.
LLM 호출 중 오류가 발생하면 사용자에게 오류를 노출하지 않고, 안내 메시지를 반환한다.
각 요청의 주요 정보(요청 식별자, 사용자 입력, 응답 생성 시간)를 로그로 남긴다.

# 프로그래밍 요구 사항
모든 기능을 TDD로 구현해 단위 테스트가 존재해야 한다.
테스트 클래스는 대상 클래스와 같은 패키지에 위치시킨다.
컨트롤러는 요청 역직렬화와 응답 반환만 담당하고, 비즈니스 로직은 서비스 계층에 위임한다.

# 힌트

## 설계
채팅 API 엔드포인트는 POST /api/chat으로, 요청 본문에 message(필수)와 sessionId(선택)를 받고, 응답에 requestId, message, durationMs를 포함한다.
sessionId는 클라이언트가 생성하는 대화 단위 식별자이고, requestId는 서버가 생성하는 요청 단위 식별자이다. 이 두 식별자는 1:n 관계이다.

## 테스트
컨트롤러 테스트는 @WebMvcTest로 웹 레이어만 로드하고, 서비스를 mock으로 대체한다.
서비스 테스트는 LLM 클라이언트를 mock하여 LLM 호출 없이 로직을 검증한다.
Spring AI는 공식 mock ChatClient를 제공하지 않으므로 Mockito를 사용한다.
