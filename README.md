# spring-ai-gift-chat

# 기능 요구 사항
선물 추천 서비스가 여러 LLM 프로바이더를 지원하도록 확장한다.

- 서비스는 여러 LLM 프로바이더를 지원하며, 설정에 따라 전환할 수 있다.
- 프로바이더를 변경하더라도 사용자에게 보이는 응답 형식은 동일해야 한다.

# 프로그래밍 요구 사항
- 서비스 계층은 구체적인 LLM 클라이언트가 아닌 추상화에 의존해야 한다.
- 프로바이더 전환 시 기존 테스트가 깨지지 않아야 한다.

# 힌트
## 설계
- LLM 프로바이더 전환은 인터페이스와 @ConditionalOnProperty로 구현한다. 프로바이더마다 지원하는 기능 범위가 다르므로, 단순 채팅만 추상화하고 고급 기능(Tool, 구조화 추출, 대화 메모리 등)은 Spring AI를 직접 사용하는 2계층 구조를 고려한다.

## 테스트
- 서비스 테스트는 LLM 클라이언트 인터페이스를 mock하여 LLM 호출 없이 로직을 검증한다.
- 프로바이더를 교체해도 서비스 테스트는 인터페이스에 의존하므로 변경 없이 통과해야 한다.
